In the original program arr[3] is evaluated before division by 0. Accessing arr[3] throws
 ArrayIndexOutOfBoundsException, so that catch block executes (not ArithmeticException).
Example program demonstrating this:
public class Q1_TestCatch {
 public static void main(String[] args) {
 try {
 int[] arr = {10, 20, 30};
 int result = arr[3] / 0; // arr[3] -> ArrayIndexOutOfBoundsException thrown here
 System.out.println(result);
 } catch (ArithmeticException e) {
 System.out.println("Division by zero!");
 } catch (ArrayIndexOutOfBoundsException e) {
 System.out.println("Array index out of range! -> This will execute");
 } catch (Exception e) {
 System.out.println("General exception!");
 }
 }
}
